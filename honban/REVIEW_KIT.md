## レビューの3レンズ（誰に何を聞くか）
- **院長レンズ**（事実・声）: 事実が正しいか／"自分の言葉に聞こえるか"
- **生活者レンズ**（家族・近所・想定患者）: 本物の町の歯医者に見えるか／冷たくないか／わかるか
- **プロレンズ**（デザイナー/歯科コピー）: 第一印象・導線・コピー・差別化

各ページ共通で、まず生活者に **「これ、AIが作ったサイトっぽい？それとも本物の医院っぽい？理由は？」** を聞くのが一番効く。

## ページ別の追加質問

### トップ（index）
- 院長: 開院予定・特徴・診療時間・あいさつの事実は正しい？
- 生活者: 何の医院か5秒でわかった？どんな人がやってるか伝わった？
- プロ: ファーストビューで誰に何を訴求してるか明確か／CV導線は？

### 通院をためらう方へ（dental-anxiety）
- 院長: 「痛かったら止める」「初回はお話だけ」、実際にこの通りやれる？言葉は自分のもの？
- 生活者: 歯医者が苦手な人として、これ読んで安心できる？空回りしてない？
- プロ: 寄り添いが具体に裏打ちされてるか／差別化になってるか

### 院長・当院（about）
- 院長: あいさつ・経歴・当院の考え、これはあなたの言葉・事実？直したい所は？
- 生活者: この院長に診てもらいたいと思った？人柄が見えた？
- プロ: 信頼構築の要素（顔/経歴/思い）が機能してるか

### 院内紹介（facility）
- 院長: 各エリアの説明は実際の院内と合う？（写真差し替え前提）
- 生活者: 清潔・落ち着く雰囲気が伝わる？
- プロ: 写真が入る前提でレイアウトは成立してるか

### 診療案内（services）
- 院長: 各診療の説明・期間目安・「削らない」の表現は適切？
- 生活者: 自分の困りごとがどれか分かった？通院回数の目安は役立った？
- プロ: 情報の過不足／SEO（辻堂×診療）／料金導線

### 料金（pricing）
- 院長: 料金の考え方・保険/自費の線引きは正しい？（価格は確定後）
- 生活者: 「いくらかかるか」の不安は減った？
- プロ: 透明性／自費の説明姿勢／医療広告の表現

### アクセス（access）
- 院長: 住所・道順・駐車場・診療時間は正しい？（確定後）
- 生活者: 場所・行き方がすぐ分かった？
- プロ: NAP一貫性／地図／ローカルSEO

### よくあるご質問（faq）
- 院長: 回答は実際の方針と合う？追加すべきQは？
- 生活者: 自分の聞きたいことがあった？足りないQはX？
- プロ: FAQの網羅性／構造化データ

### お知らせ（news）
- 院長: 内覧会・開院・求人の実予定は？
- 生活者: 「ちゃんと開院に向けて動いてる」感じがする？
- プロ: 更新頻度の設計／記事ページの要否

## 自動ツール（客観・無料）— 各ページに実行
1. **Lighthouse**（Chrome DevTools → Lighthouse タブ）: Performance / Accessibility / Best Practices / SEO の4スコア。90以上を目標
2. **PageSpeed Insights**（pagespeed.web.dev）: ライブURLを入力。モバイル速度・Core Web Vitals
3. **Googleリッチリザルトテスト**（search.google.com/test/rich-results）: 構造化データ（Dentist/FAQPage）の検証
4. **WAVE**（wave.webaim.org）: アクセシビリティ自動チェック（コントラスト・alt・見出し）
5. **医療広告ガイドライン セルフチェック**: 誇大/断定/体験談/ビフォーアフターが無いか（厚労省ガイドライン）
