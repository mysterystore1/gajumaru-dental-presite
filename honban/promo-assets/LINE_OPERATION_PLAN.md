# 公式LINE 運用設計（辻堂がじゅまる歯科）

> 予約＝Airリザーブ／接点・リコール・アフター＝公式LINE の二段構え。
> **医療広告GL・ステマ規制を厳守**（レビュー依頼NG・体験談NG・断定/最上級NG・自費はリスク/費用併記）。
> ブランド：やさしい・家族・歯医者が苦手な人に寄り添う・がじゅまる猫。開院 2026/8/1。

---

## 0. 役割分担（重要）

| 機能 | 担当 | 補足 |
|---|---|---|
| 予約受付・変更・空き枠 | **Airリザーブ** | 予約ページURL/QRを発行 |
| 患者接点・リマインド・リコール・アフター | **公式LINE** | 配信1通1〜3円・開封率高 |
| 連携方法 | LINE → Airリザーブ | リッチメニュー/自動応答に**予約ページURLを貼る**（LINE内予約完結はしない） |

※Airリザーブのリマインドは有料プラン（ベーシック5,500円〜）。**リマインドはLINEに寄せると安い**。

---

## 1. リッチメニュー（6分割・常時表示）

```
┌───────────┬───────────┬───────────┐
│ ① ご予約   │ ② 診療時間  │ ③ アクセス  │
│ (Air予約)  │ ・休診日    │ ・地図/駐車場│
├───────────┼───────────┼───────────┤
│ ④ 初めての方 │ ⑤ よくある質問│ ⑥ 電話する  │
│ ・WEB問診  │ (FAQ)      │ (tel発信)  │
└───────────┴───────────┴───────────┘
```

- ①＝Airリザーブ予約ページURL（最重要・左上固定）
- ②③＝サイトの該当セクションURL or LINE内テキスト返信
- ④＝初診の流れ・持ち物・WEB問診（あれば）。「いきなり削らない／まず話を聞く」方針を一言
- ⑤＝サイトFAQへ
- ⑥＝`tel:` で電話発信
- デザイン：生成り＋テラコッタ＋がじゅまる猫アイコン（サイト/チラシと統一）

## 2. 友だち追加の入口（開院前から仕込む）

- 受付POP／診察券／会計時カード／チラシ／サイト／内覧会／GBP
- QRは Airリザーブ予約QR と **別**（LINE友だち追加QR）。混同しない
- 来院後の**アフター・リコールをLINEで受け取れる**ことを追加メリットに（割引等の対価は付けない＝ステマ規制）

### あいさつメッセージ（友だち追加直後・自動）
```
友だち追加ありがとうございます。辻堂がじゅまる歯科です🌿
ご予約はメニュー①から、診療時間・アクセスは②③からご確認いただけます。
診療やお口のことで気になる点があれば、このままメッセージでもどうぞ。
※このLINEは予約確認・検診のご案内などにも使います。
```

## 3. 自動応答（キーワード）

| キーワード例 | 返信 |

| 予約／よやく | Airリザーブ予約URL＋「変更・キャンセルも同じページから」 |

| （その他） | 「担当が確認しご返信します。お急ぎはお電話で」 |

## 4. 配信シナリオ（ステップ配信）

> すべて**診療フォロー/案内**の範囲。レビュー依頼・体験談・断定はしない。

### A. 初診後フォロー（来院フォロー）
- **当日〜翌日**：「本日はご来院ありがとうございました。麻酔後の食事・痛み止め等の注意点。気になることはこのままご返信を」
- **3〜4日後**：「その後、痛み・違和感などございませんか。気になる点があればご連絡ください」← レビュー文言は入れない

### B. 治療完了後
- 「治療おつかれさまでした。きれいな状態を保つため、◯か月後の定期検診をおすすめします。時期にこちらからご案内します」

### C. リコール（定期検診の再来）★リピートの本丸
- **3か月後／6か月後**（処置種別でセグメント）：
  「前回から約◯か月です。検診・クリーニングのおすすめ時期になりました。ご予約はこちら（Airリザーブ）」
- ホワイトニング後・歯周治療後など**処置別に文面を最適化**（一律より反応2〜3倍）

### D. キャンセル枠・お知らせ（低頻度）
- 急な空き枠、年末年始/GW/お盆の休診、季節の健康トピック（むし歯予防・乳歯ケア等の一般情報）
- 配信は**月1〜2回まで**（多いとブロック増）

---

## 5. セグメント設計（最小で開始）

- 最終来院日（3か月/6か月経過）
- 処置種別（予防/ホワイトニング/歯周/小児/矯正）
- 家族属性（子育て世帯/シニア）※任意
- 開始は「最終来院日」だけでも十分。Lステップ等の高度自動化は軌道に乗ってから

## 6. 🚫 GL・ステマ厳守ルール（運用者向け・最重要）

**やらないこと（一発アウト）**
- ❌ 「Googleに口コミ/レビューお願いします」の依頼配信（医療広告GL：口コミ依頼は誘引性で禁止）
- ❌ レビュー・紹介の**見返りに割引/プレゼント**（2025/3 歯科で消費者庁措置命令・景表法/ステマ）
- ❌ **満足した人だけ**にレビュー誘導（レビューゲーティング・Googleポリシー＋景表法違反）
- ❌ 患者の感想・体験談をLINE配信やサイトに転載
- ❌ 「絶対治る」「痛くない」「地域No.1」等の断定・最上級
- ❌ 自費（ホワイトニング/矯正/インプラント等）を**費用・リスク併記なし**で勧める配信

**やってよいこと**
- ✅ アフターの体調確認・注意点案内（診療フォロー＝広告でない）
- ✅ 定期検診/クリーニングの**案内**（依頼でなく案内）
- ✅ 休診/空き枠/一般的な健康情報
- ✅ 口コミは**患者の自発に委ねる**（誘引しなければ広告に当たらず合法）。どうしても声を集めるなら「ご意見・ご感想はこちら」を**私的フィードバック窓口**（LINE返信/フォーム）に留め、公開レビューへ誘導しない

## 8. KPI（見るだけ・盛らない）

- 友だち数／ブロック率（配信頻度の指標）
- リコール配信→予約の到達率
- 定期検診の再来院率（本命指標）
- ※「口コミ件数」をKPIにしない（依頼インセンティブの誘惑を断つ）

---

## 9. 論点メモ

- ツール：まずは**LINE公式の標準機能**（あいさつ/リッチメニュー/自動応答/セグメント配信/配信予約）で開始。Lステップ等の有料拡張は再来導線が固まってから。
- Airリザーブ有料化の判断：LINEでリマインドを賄えば、Airは**フリー〜ベーシック**で足りる可能性。空き枠連携・複数メニューが要るならスタンダード検討。
- 文面の最終確認：自費・効果に触れる配信は**配信前にGLセルフチェック**（断定/最上級/体験談/費用リスク欠落の4点）。
