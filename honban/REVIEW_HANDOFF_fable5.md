## 3. 分析してほしい観点

依頼者が特に気にしている点 → **「他のよくできた歯科サイトはCSSを駆使して出来栄えが高い。それらと比べて視覚的クオリティで見劣りしないか。文言の温度は適切か。」**

- [ ] **視覚的クオリティ全般** — 第一印象、洗練度、情報の階層、余白設計、タイポグラフィ、配色、画面の"密度と抜け"。プロが作った歯科サイトと並べたときに見劣りする要素はどこか。
- [ ] **CSS技法（最重要）** — 出来栄えを底上げできる、まだ使えていない/活かせる現代的CSS。例: グラデ・影の質、`clamp()` 流体タイポ、Grid応用、`:has()`、スクロール連動アニメ／マイクロインタラクション、`prefers-reduced-motion` 配慮、画像の質感演出、セクション境界の有機的処理。**具体的なコード例つきで**。
- [ ] **文言（コピー）** — 固さ・AI臭・テンプレ感・温度。患者目線になっているか。改善できる箇所の **before/after 書き換え案**。
- [ ] **UX/導線** — 予約・電話への動線、グローバルナビ、情報の見つけやすさ、CTAの強さ、フォーム。
- [ ] **競合歯科サイトとの差** — よくできた歯科サイトが使っている表現・構造・演出で、本サイトに足りていないもの。
- [ ] **レスポンシブ / アクセシビリティ / 表示速度** — モバイル表示、フォーカス可視・コントラスト・alt、フォント読み込み・画像最適化。
- [ ] **コード/設計品質（任意）** — CSS設計（命名・重複・カスケード）、共通部品運用、保守性。

## 4. 前提条件・制約（レビュー時に踏まえてほしい）

- **「データが仮であること」自体への指摘は不要**。実写真・実NAP・院長の言葉は後で差し替え前提。**コード・デザイン・文言"構造"で改善できる点**を見たい。
- 実写真はプロ撮影後に差し替え予定（現状はプレースホルダ）。画像の"中身"でなく**見せ方・枠・演出**を評価対象に。
- **医療広告ガイドライン遵守**: ビフォーアフター・患者の声・症例・捏造実績・体験談は載せない方針。これらを増やす提案は不可。
- **世界観「暖色×木」は維持**（海/青系は同町競合と被るため不採用）。配色提案はこの範囲で。
- **四角カードで情報を囲わない（罫線＋余白型）方針**。ただし、この方針自体が視覚品質を損ねていると判断するなら、その指摘・代替も歓迎（妥当性ごと問うてよい）。
- **フレームワーク/ビルドツールは原則入れない**（素のHTML/CSS/JS）。導入を勧める場合は理由とコストを明記。
- 既存の社内資料（`honban/SITE_REVIEW.md` / `COMPETITOR_RESEARCH.md` / `COPY_DESIGN_RESEARCH.md` / `REVIEW_KIT.md`）は参考にしてよいが、**結論に引きずられず独立に評価**してほしい。
