# CLAUDE.md — 辻堂がじゅまる歯科 サイト刷新

このリポジトリで作業する際の絶対規約。ユーザー指示と矛盾する場合は作業を止めて確認する。

## プロジェクト概要

2026年8月1日開院の歯科医院サイト。現行の静的サイト（/honban/）を、Claude Designで確定したデザインに基づきフルリビルドする。医療広告ガイドライン準拠のコンテンツ資産（自費診療のリスク・費用記述等）は既に水準以上であり、**リビルドで劣化させないことが最優先**。

## 1. データの単一情報源（SSOT）

- 人間用の正本：`CLINIC_FACTS.md`／機械用ミラー：`_data/site.json`
- 医院名・電話番号・住所・診療時間・休診日・診療内容・ナビラベルを **site.json 以外にハードコードすることを禁止**。テンプレート内では必ず参照で記述する
- 既存コードにハードコードを発見したら参照へ置換する
- 値の変更手順：CLINIC_FACTS.md を先に更新 → site.json に同期 → `_meta.lastSyncedWithFacts` を更新
- site.json の `null` は「未確定」。**仮値を創作して埋めることを厳禁**。サイト上は「準備中」系の表記で受ける

## 2. 表記ルール

- 医院名：ひらがな「がじゅまる」。カタカナ表記禁止
- 時刻：半角コロン・半角ハイフン「10:30-13:30」、時間帯区切り「 / 」（半角スラッシュ前後半角スペース）
- titleタグ・og:title の区切りは全角「｜」で統一し、両者の文言を一致させる
- ナビラベルは site.json の定義と完全一致（ヘッダー・フッター・パンくず・本文リンクすべて）
- 未確定情報には必ず「（仮）」を付す。（仮）なしの仮値は禁止

## 3. 医療広告ガイドライン（違反はリリースブロッカー）

- 効果の保証・断定表現の禁止（「必ず治る」「痛くない」等）。「〜を目指す」「痛みに配慮」等の表現に留める
- ビフォーアフター写真・比較優良表現（地域No.1等）・誇大表現の禁止
- 自由診療（ホワイトニング・マウスピース矯正・インプラント・スポーツマウスガード）のページでは、**治療内容・標準的な費用・治療期間/回数・主なリスクと副作用の記述を削除・省略しない**（限定解除要件）。リファクタ時もこの4点の残存を必ず確認
- 出典のない統計数値をコピーに入れない

## 4. design-reference/ の扱い

- `design-reference/` は Claude Design 成果物の**参照専用**ディレクトリ。中のコードを本番へコピーすることを禁止
- 視覚の正：採用案のスクリーンショットと `design-tokens.css`
- 実装は本プロジェクトのテンプレート構造で行い、見た目のみ design-reference に寄せる

## 5. デザイントークン

- 色・余白・フォントサイズ・角丸・影は `design-tokens.css` の CSS変数のみ使用。生値のハードコード禁止
- スペーシングは8px基準スケール、タイプスケールは7段階以内、角丸3種以内、影2種以内
- 装飾SVGは `assets/illust/` のスタイルガイド（illustration-styleguide.md）準拠。色はCSS変数参照に置換する

## 6. アーキテクチャ上の注意（既存決定の上書き）

- DECISIONS.md の **DEC-002（ビルドツール非採用）・DEC-004（共通部品コピー方式）は本リビルドで廃止**される。共通部品のコピペ運用がページ間ドリフト（料金ページのダミー電話等）の根本原因だったため、SSG（ビルド時include）＋site.json参照へ移行する
- この上書きは新しい DEC エントリとして DECISIONS.md に追記してから実装に着手すること（このリポジトリの意思決定ログ文化を維持する）
- 本番昇格手順は DEC-006 を踏襲しつつ、公開時チェックリスト（下記8）を追加

## 7. 作業規律

- 1セッション=1フェーズ=1ブランチ。フェーズをまたぐ大規模変更を1コミットに混ぜない
- コンテンツ移植は `parity.md`（現行全ページの本文・自費リスク記述・NAP・FAQの抽出リスト）と突合して行う。parity.md に無い削除・要約を勝手に行わない
- 削除系の操作（ページ・セクションの除去）は理由をコミットメッセージに明記

## 8. 完了条件（機械検証）

リリース前に以下を全てパスすること：

```
grep -rn "（仮）" <公開対象>        # ヒット0件（公開時）
grep -rn "tel:" <公開対象>          # 全て site.json の phoneTel と一致、X を含まない
grep -rn "0465\|XX-XXXX" <公開対象> # ヒット0件（誤番号・ダミーの残存チェック）
grep -rn "ガジュマル" <公開対象>     # ヒット0件（カタカナ表記禁止）
```

加えて公開切り替え時：noindex解除 → canonical確認 → og:image の実在をURL直叩きで確認 → sitemap送信 → 旧URLの301 → スマホ実機で全ページ回遊。

## 9. 機密の取り扱い

- 本リポジトリは公開されている。内部ドキュメント（HANDOFF・OWNER_DECISIONS・人物情報等）の分離方針が確定するまで、**新たな個人情報・経営情報をこのリポジトリに追加しない**
