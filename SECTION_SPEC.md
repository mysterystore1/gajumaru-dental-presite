## FV（ヒーロー）仕様

- 構造: 写真 → 4段スクリム＋メルト層（最下部を--bgへ溶かす）→ 予約カード。写真の上に本文ボタンを置かない
- 序列（SP）: ①差別化3バッジ（ヒーロー帯内・「土日も診療｜平日は19:00まで」等）→ ②予約カード → ③リボン導線2本
- 重なりの整列: 見出し=下128px／バッジ=下72px／カード=−40px（8px系）。z順は バッジ < カード
- 予約カード内序列: WEB予約（主・--cta塗り）→ 電話（枠線）→「まずは相談だけしたい方へ→」（テキストリンク）
- 開院日ラベル「2026年8月1日(土)開院｜藤沢市辻堂」をスクリム帯内に配置
- PC: 予約カードをヒーロー右に配置（ミニ診療時間つき）

## refined 表現ルール

- 見出し装飾: 「筆致ライン＋葉1枚」1種のみ、全セクション共通（--orn-heading）
- 波形区切り: 1形状のみ。--bg-alt帯の上下とフッター境界に限定（--wave-sp/pc）
- 装飾の物語配置: 上から 足あと → 鳥 → 足あと → 気根 → 猫（CTAで座る）。1セクション1点厳守
- 主要写真: --line色1px・6pxオフセット枠（--frame-offset）
- マイクロ: 情報ラベル先頭に4pxドット、区切り線は両端フェード

## 実装注記（重要）

1. **データ**: デザイン内の電話番号「0466-XX-XXXX（仮）」・休診曜日（木曜=誤り）・診療時間はすべて破棄し、site.jsonから流す。実際の休診は火・祝
2. **時刻表記**: デザインは全角ダッシュ「9:00–13:00」。実装はCLINIC_FACTS準拠の半角ハイフン「10:30-13:30」に正規化
3. **PC予約カードのスクロール挙動**: デザインでは未定義（キャンバスの静止画）。実装方針=**ヒーロー内静的を基本**とする。追従(sticky)にする場合は、ヒーロー通過後にミニ診療時間表を畳んでボタン2本の縮退版に切り替えること（PC右固定要素の視線競合レビュー指摘への対策）
4. **WEB予約URL**: site.jsonでnullの間、予約カードの主ボタンは「WEB予約（準備中・◯月◯日受付開始）」表記とし、リンク先は電話・フォーム誘導へ。全ページ同一文言（現行サイトの文言不統一の再発防止）
5. **アニメーション**: 入れる場合は opacity+translateY(12px)、600ms、ease-out、初回のみ、prefers-reduced-motion:reduceで全停止（Fable5レビュー処方箋）。opacity:0初期値はJS無効環境のフォールバック必須
6. **医療広告**: 「平日は19:00まで」等の事実系バッジはsite.json hoursと矛盾しないこと。効果保証系の文言をコピー調整フェーズで混入させない
